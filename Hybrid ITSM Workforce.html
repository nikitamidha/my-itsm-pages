TechGuard ITSM Hybrid Workforce Org Chart

TechGuard Service Operations Hybrid ITSM workforce: humans manage judgment, trust, governance, physical work, and exceptions; autonomous agents run the repeatable operational work.
ITForce Control Plane Salesforce Service Cloud + Agentforce + Field Service Lightning + audit trail, work orders, and client communications.
Clients and Managed Assets 15,000 B2B clients; CCTV cameras, web hosting assets, network circuits, sites, contacts, SLAs, and telemetry.
L1 Frontline Service Client-facing command layer for intake, trust, comms, and relationship-aware decisions.
CSA Leads Own queue health, escalations, client-sensitive accounts, and frontline performance.
L1 Customer Service Agents Approve outbound comms, handle ambiguous intake, manage client expectations, and close the loop.
Autonomous L1 workforce
Scoping & Triage Agent Classifies service line, priority, scope, and impact.
Router Agent Routes by confidence, skill, SLA, tier, and incident type.
Communication Agent Drafts client updates, closure notes, and legal-sensitive notices.
SLA Prediction Agent Warns before breach and prioritizes human attention.
Gate: CSA Approval All outbound B2B client communications require human approval.
Autonomous Platform Layer Always-on digital workforce watching the full fleet before a client reports pain.
Monitoring & Detection Agent Opens structured incidents proactively from telemetry and health signals.
Asset Discovery & Inventory Agent Finds unmanaged, missing, new, or changed assets and drafts registry updates.
Preventive Maintenance Agent Schedules firmware, inspections, failover tests, and backup verification.
Impact & Correlation Agent Clusters incidents by ISP, provider, region, asset model, or root cause.
Gate: Cluster Review CSA spots surface patterns; IT Ops validates technical correlation and response.
Backend IT Operations Technical authority for risky actions, specialist depth, autonomy governance, and major incidents.
IT Ops Lead Owns major incidents, calibration, autonomy thresholds, and governance.
IT Ops Managers Approve risky changes, validate diagnostics, reroute novel cases, and authorize dispatch.
Human specialists
Web Hosting Specialist Checkout, rollback, malware, DB, production risk.
Network Specialist ISP, SD-WAN, routing, brownout, demarc evidence.
CCTV / VMS Specialist NVR, PoE, storage, firmware, footage gaps.
Security / Compliance Specialist Legal notices, malware, audit, compliance risk.
Specialist autonomous agents
Web Hosting Resolution Agent Safe fixes, rollback proposal, DNS, cache, PHP.
Network Diagnostic Agent Blackout vs brownout, OSI layer, ISP evidence.
CCTV Diagnostic Agent Camera vs switch, PoE cycle, remote attempts.
Footage / Deploy / Baseline Agents Recording health, deploy regression, circuit baseline.
Gate: IT Ops Approval Irreversible changes, ISP escalation, dispatch, fleet blast radius, and legal/compliance risk.
Field Service Physical authority for site diagnosis, repair, install, access realities, and customer sign-off.
Field Service Lead Owns territory capacity, dispatch quality, parts readiness, and repeat-visit prevention.
Field Service Agents Execute physical diagnosis, replacement, cabling, installation, verification, and site sign-off.
Autonomous field workforce
Field Dispatch Agent Creates FSL WorkOrder with root cause, parts, access, warranty, prior attempts, and checklist.
Field Service Copilot Guides on-site steps, captures voice/tap notes, surfaces similar cases, and enforces close criteria.
Gate: Assist / Scope Change Field agent escalates when on-site reality differs from the briefing or remote action is needed.
Proactive Path Assets -> Monitoring -> Triage -> Service-line agent -> safe fix or IT Ops approval.
Reactive Path Client -> CSA + Triage -> Router -> autonomous diagnostic or specialist queue.
Specialist Path Agent selects the right human specialist by skill, permission, confidence, and availability.
Field Path IT Ops approves dispatch -> Field Dispatch Agent -> Field Agent + Copilot.
Closure Path Verify checklist -> Comms draft -> CSA approval -> client update -> case close.

Read vertically as hierarchy and horizontally as operating model: green AI workers handle repeatable sensing, triage, diagnosis, routing, drafting, and guidance; blue human workers own trust, judgment, approvals, specialist depth, and physical work.
